# Unit Test

This repository contains unit tests for MSIS-NN functions, based on the [Unity](https://www.throwtheswitch.org/unity) test framework.  

## File Structure  

| Directory  | Description  |
|------------|-------------|
| `Scripts`  | Contains scripts for generating test data and test runners. |
| `TestCases` | Subdirectories prefixed with `test_` contain individual test cases. |

## Requirements  

To run the unit tests, the following dependencies are required:  

- **Nuclei SDK** (master branch, version ≥ 0.7.0)  
- **Nuclei RISC-V GNU Toolchain** (version ≥ 2024.06)  
- **Nuclei QEMU** (version ≥ 2024.06)  
- **[Ruby](https://www.ruby-lang.org/zh_cn/)** (any recent version; tested with 2.7.0, required for generating test runners)  
- **(Optional) Python 3** (version ≥ 3.11) along with the required dependencies specified in:  
  - `<NMSIS>/NMSIS/NN/Tests/UnitTest/Scripts/requirements.txt`  

## How to Run  

### Generating Test Runners  

To generate test runners, run the following command:  

```shell
cd /path/to/UnitTest
python3 Scripts/generate_test_runner.py
```  

This script scans all test cases and generates corresponding test runners.  

For example, the test case `riscv_avgpool_s8` will have its generated test runner located at:  

```shell
.
├── Makefile
├── test_riscv_avgpool_s8.c
└── Unity
    ├── TestRunner                               (autogenerated)
    │   └── unity_test_riscv_avgpool_s8_runner.c (autogenerated)
    └── unity_test_riscv_avgpool_s8.c
```  

### Running Tests  

To run a test case with a specified Nuclei CPU `CORE` and `ARCH_EXT`, use the `make` command.  

For instance, to test with `CORE=n3000` and `ARCH_EXT=_xxldsp`, execute:  

```shell
cd /path/to/testcase
make CORE=n3000 ARCH_EXT=_xxldsp SIMU=qemu clean all run_qemu
```  

You can modify `CORE` and `ARCH_EXT` to test different configurations.  

### (Optional) Regenerating Test Data  

**Note**: Test data is tracked in Git, so regenerating it is optional.  

The test parameters for each test case are defined in [test_plan.json](Scripts/test_plan.json).  

Test data generation requires a schema file, [schema.fbs](Scripts/schema.fbs), which is identical to the [schema.fbs](https://github.com/tensorflow/tflite-micro/tree/main/tensorflow/compiler/mlir/lite/schema/schema.fbs) used in the *TensorFlow Lite Micro* repository.  

To regenerate test data, make sure you have installed the Python dependencies specified in `<NMSIS>/NMSIS/NN/Tests/UnitTest/Scripts/requirements.txt` and run:  

```shell
cd /path/to/UnitTest
python3 Scripts/generate_test_data.py
```  

For additional options, use:  

```shell
python3 Scripts/generate_test_data.py -h
```
